PhotoStore

How we built a cloud photo app using plain-English prompts in Cursor — told for anyone, no coding background required.

1

What did we build?

PhotoStore is a web app where people can upload and view photos. The files live securely in Amazon Web Services (AWS) — like a private locker in the cloud.

For signed-in users

Create an account, sign in, upload photos, and see your own gallery.

For visitors (guests)

Try the app first — upload up to 2 photos before signing up. When they create an account, those photos move to their account.

Behind the scenes

Small programs in the cloud handle uploads, security, and storage — we did not have to wire all of that by hand.

One project folder

Website and cloud backend now live together in one repository (a monorepo), so the team works in one place.

2

What is Cursor?

Cursor is a code editor with an AI assistant built in. Instead of writing every line yourself, you describe what you want — the assistant reads the project and makes changes, explains them, and runs commands when needed.

You type a prompt → AI reads the codebase → Code & docs update → You review & deploy
  • Prompt — your instruction in everyday language
  • Outcome — what actually changed (features, docs, fixes)
  • Iteration — follow-up prompts refine the result (like editing a draft)

Vivek's prompts — Getting it on AWS

First goal: run the project on a real AWS account.

Prompt
can you follow all steps for me to deploy this project on my aws account
What happened

The assistant walked through prerequisites (AWS CLI, Node, Yarn), deployed the cloud stack, and connected the pieces so the API and login system were live in your account.

Follow-up prompt
do I need this in FE code CognitoIssuer
What happened

Clarified what the React app actually needs from deploy outputs — keeping the front end simple and avoiding unused configuration.

Follow-up prompt
run serverless remove
What happened

Tore down the cloud stack safely (including emptying the storage bucket first so removal could complete). Useful for cleanup or starting fresh.

Vivek's prompts — Documentation & diagrams

So the team (and future readers) understand how it works.

Prompt

What happened

Expanded the main README to cover recent work (guest photo upload, environment sync after deploy, merge-on-sign-in), API tables, and Mermaid diagrams (flowcharts and sequences) so non-coders can follow the guest → sign-in → merge journey visually.

Vivek's prompts — One repository for everything

Prompt

What happened

Moved the React website into packages/client and the cloud API into packages/api, set up Yarn 4 workspaces, and unified commands like yarn dev and yarn deploy:sync from the project root.

Later prompt
can you pull changes and resolve conflicts
What happened

Synced with remote Git changes and merged overlapping edits so both teammates' work could land in one branch.

Follow-up prompt

What happened

Rewrote README paths, folder trees, and commands for the monorepo layout so documentation matched reality.

What we learned

  • Start with outcomes — “deploy on my AWS,” “guest can upload 2 photos,” not low-level code requests
  • Follow up in plain language — “put that in the README,” “add logs everywhere,” “fix the readme for the new folders”
  • Let the AI handle glue — monorepo moves, merge conflicts, env sync scripts
  • Document for humans — diagrams and README sections matter as much as the code
